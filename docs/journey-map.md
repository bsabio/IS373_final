# Customer Journey Map - MyWebClass.org

## Quick Reference: Student Submission Journey

1. **Browse gallery** - Discover existing design style examples
2. **Choose a design style** - Select style to implement (Bauhaus, Minimalism, etc.)
3. **Build demo** - Create authentic website using style principles
4. **Submit URL and explanation** - Fill form with demo link, screenshot, and authenticity writeup
5. **Instructor reviews in CMS** - Professor evaluates submission in Sanity
6. **Approved work appears publicly** - Submission goes live in gallery for others to learn from

---

## Key Journey Insights

### Common Pain Points Across Journeys
1. **Lack of status visibility** - Users don't know submission state
2. **No feedback mechanism** - One-way communication
3. **Mobile experience gaps** - Heavy desktop assumption
4. **Unclear timelines** - No expectations set

### Critical Touchpoints
1. **Submission form** - Must be simple, clear, with validation
2. **Review dashboard** - Needs efficiency features for instructors
3. **Style demo pages** - Core educational value
4. **Email notifications** - Keep users informed

### Emotion Trajectory Goals
- **Students:** Uncertain ‚Üí Confident ‚Üí Proud
- **Instructors:** Neutral ‚Üí Efficient ‚Üí Satisfied
- **Hobbyists:** Curious ‚Üí Enlightened ‚Üí Confident

### Success Metrics by Journey
- **Student:** Submission approved within 72 hours
- **Instructor:** Review 15 submissions in < 30 minutes
- **Hobbyist:** Return visit within 7 days
